## 故障排除

### 问题: API 调用失败

**解决方案**:
```
检查 api-config.json 配置:
- API endpoint 是否正确
- API key 是否有效
- 网络连接是否正常
```

### 问题: Web Search 超时

**解决方案**:
```
在 api-config.json 中调整:
- 减少 maxTopics 数量
- 增加 delayBetweenRequests 延迟
- 减少 resultsPerTopic 数量
```

### 问题: 评分偏低

**解决方案**:
```
在 api-config.json 中调整:
- 修改 scoring 的 min/max 值
- 调整 excellentThreshold 和 goodThreshold
```

## 最佳实践

1. **定期运行**: 建议每天或每周定期生成报告
2. **配置优化**: 根据需求调整 API 配置和评分阈值
3. **数据积累**: 保存历史报告用于趋势分析
4. **分享报告**: HTML 报告可轻松分享给团队成员
5. **反馈迭代**: 根据使用反馈优化评分标准
